```
├─cmd：目录用于存放配置文件和配置相关的代码。
├─config：目录用于存放配置文件和配置相关的代码。
├─docs：目录包含了项目的文档和设计文档。其中，api 目录用于存放 API 文档。
├─hack：目录用于存放生成models的配置文件。
├─internal：目录是项目的主要实现目录，包含了业务逻辑的实现。
│  ├─app：存放应用程序的主要业务逻辑代码。
│  │  ├─constants：系统常量信息。
│  │  ├─controllers：按照业务模块划分的控制器代码。
│  │  ├─db：目录可能用于存放数据库相关的代码，如数据库连接、数据库操作等
│  │  ├─dto：目录用于存放数据传输对象（DTO）的定义，用于处理控制器层和服务层之间的数据传递。
│  │  ├─middlewares：目录用于存放中间件代码。
│  │  ├─models：用于存放模型定义的代码。这里通常包含数据库表的映射对象（如结构体）以及与数据库操作相关的接口和实现。
│  │  │  ├─association：用于存放处理模型之间关联关系的代码，例如一对一、一对多和多对多关系。
│  │  │  ├─model：存放具体的模型定义。(程序自动生成）
│  │  │  ├─query：存放与查询相关的代码，如查询构建器和分页器等。(程序自动生成）
│  │  ├─repositories：目录用于存放数据仓库的实现代码。
│  │  ├─services：目录用于存放服务层的实现代码。
│  │  └─validators: 存放输入验证相关的代码。
│  ├─pkg：存放通用的、可复用的代码。
│  │   ├─database：数据库相关的代码。
│  │   ├─logger：日志记录器相关的代码。
│  └─system：系统的工具函数或帮助类。
└─pkg：存放通用的、可复用的代码。
├─migrations：目录用于存放数据库迁移文件，包括升级和回滚脚本。
├─routers：目录用于存放路由设置相关的代码。
│  ├─paths：用于存放这些路由路径常量。
└─web：目录用于存放与 Web 相关的代码。
├─handlers：目录用于存放处理请求的处理函数。
├─static：目录用于存放静态资源文件，例如 CSS、JavaScript 等。
└─templates：目录用于存放模板文件，例如 HTML 模板。
```
